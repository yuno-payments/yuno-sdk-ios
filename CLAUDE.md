# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is the **public-facing iOS SDK** for [Yuno Payments](https://www.y.uno/) -- a fast checkout solution that integrates companies into the payments and fraud ecosystem (Checkout + Routing + Reconciliations + Alternative Payment Methods). The SDK itself is distributed as a **pre-built binary XCFramework**; this repository contains the distribution manifests (CocoaPods podspec, Swift Package Manager `Package.swift`), two example/demo applications (UIKit and SwiftUI), and headless SDK documentation. No SDK source code is committed here -- the XCFramework binaries are attached to GitHub Releases.

The current version is **2.11.3**. The minimum deployment target is **iOS 13.0** (podspec) / **iOS 14.0** (usage documentation). The SDK is written in Swift and requires `BUILD_LIBRARY_FOR_DISTRIBUTION`.

## Essential Commands

### Development (Example App - UIKit)
```bash
cd Example
pod install
open YunoSDK.xcworkspace   # generated by CocoaPods
```
Build and run the `YunoSDK_Example` scheme in Xcode targeting an iOS simulator or device.

### Development (Example App - SwiftUI)
```bash
open YunoSwiftUI/YunoSwiftUI.xcodeproj
```
The SwiftUI example pulls YunoSDK as a package dependency. Build and run the `YunoSwiftUI` scheme.

### Testing
The UIKit example contains a basic test target using **Quick** and **Nimble** at `Example/Tests/Tests.swift`. Tests are placeholder/sample tests; there is no dedicated test suite for the binary SDK.
```bash
# From the Example workspace after pod install:
xcodebuild test -workspace Example/YunoSDK.xcworkspace -scheme YunoSDK-Example -destination 'platform=iOS Simulator,name=iPhone 15'
```

### Linting & Code Quality
```bash
pod lib lint YunoSDK.podspec --allow-warnings
```
No SwiftLint or SwiftFormat configuration is present in this repository.

### Building / Releasing
Releases follow a CI branch naming convention (`CI/release-flow/<version>`). A release consists of:
1. Updating the version in `Package.swift` (binary URL + checksum) and `YunoSDK.podspec`.
2. Tagging the commit (e.g., `2.11.3`).
3. Attaching two XCFramework ZIP artifacts to the GitHub Release:
   - `YunoSDK.xcframework.zip` (CocoaPods)
   - `YunoSDK_SPM.xcframework.zip` (Swift Package Manager)
4. Running `pod trunk push YunoSDK.podspec`.

### Key Components (Public SDK API)

The SDK exposes its API through the `Yuno` namespace. Key types and protocols:

- **`Yuno.initialize(apiKey:config:)`** -- SDK initialization (call in AppDelegate or SceneDelegate)
- **`YunoConfig`** -- Configuration: `cardFormType`, `appearance`, `saveCardEnabled`, `keepLoader`
- **`Yuno.Appearance`** -- UI customization (fonts, colors, buttons, checkboxes)
- **`YunoPaymentDelegate`** / **`YunoPaymentFullDelegate`** -- Payment flow delegate protocols
- **`YunoEnrollmentDelegate`** -- Enrollment flow delegate protocol
- **`PaymentMethodSelected`** -- Protocol for selected payment method (vaultedToken + paymentMethodType)
- **`Yuno.startCheckout(with:)`** -- Start checkout session
- **`Yuno.getPaymentMethodViewAsync(delegate:)`** -- Async method returning a UIView with payment methods
- **`Yuno.startPayment(showPaymentStatus:)`** -- Start full payment
- **`Yuno.startPaymentLite(with:paymentSelected:showPaymentStatus:)`** -- Start lite payment
- **`Yuno.continuePayment(showPaymentStatus:)`** -- Complete payment after 3DS/additional steps
- **`Yuno.hideLoader()`** -- Dismiss persistent loader
- **`Yuno.enrollPayment(with:showPaymentStatus:)`** -- Start enrollment flow
- **`Yuno.receiveDeeplink(_:)`** -- Handle URL scheme callbacks
- **`Yuno.Result`** -- Enum: `.reject`, `.succeeded`, `.fail`, `.processing`, `.internalError`, `.userCancelled`

**Headless SDK** (for custom UI):
- **`Yuno.apiClientPayment(countryCode:checkoutSession:)`** -- Returns `YunoPaymentHeadless`
- **`Yuno.apiClientEnroll(country_code:customer_session:)`** -- Returns `YunoEnrollHeadless`
- **`YunoPaymentHeadless.generateToken(data:)`** -- Generate OTT from card/vaulted token
- **`YunoEnrollHeadless.continueEnrollment(data:)`** -- Enroll and get vaulted token

## Key Patterns

- **Binary distribution only**: The SDK ships as a pre-built XCFramework. There is no compilable Swift source for the SDK itself in this repo. Changes to distribution update `Package.swift` and `YunoSDK.podspec`.
- **Delegate-based architecture**: Payment and enrollment flows use delegate protocols (`YunoPaymentDelegate`, `YunoEnrollmentDelegate`) for callbacks. The delegate provides session IDs and country codes and receives results.
- **Three integration levels**: Full (SDK-managed UI), Lite (merchant-selected payment method), and Headless (merchant-managed UI with SDK token generation).
- **Combine usage**: Example apps use Combine publishers for reactive text field binding and state management.
- **Deep link handling**: The SDK requires `Yuno.receiveDeeplink(url)` in `application(_:open:options:)` for payment provider redirects.
- **Version synchronization**: When releasing, the version must be updated in both `Package.swift` (URL path + checksum) and `YunoSDK.podspec` (version string).
- **Localization**: The SDK supports English (EN), Spanish (ES), and Portuguese (PT) via resource bundles embedded in the XCFramework.

## Important Notes

- **No CI configuration** is checked into the repo (no GitHub Actions, Bitrise, or Fastlane configs). Releases appear to be managed via an external CI system using the `CI/release-flow/<version>` branch convention.
- **Backward compatibility**: The podspec declares iOS 13.0 minimum, but README usage docs state iOS 14.0. When in doubt, target iOS 14.0+.
- **Static framework**: The podspec sets `static_framework = true`. Integrators must use `use_frameworks!` in their Podfile.
- **No source code to modify**: Since the SDK is a binary, PRs to this repo typically involve documentation updates, example app changes, or version bumps for new releases.
- **Symlink**: The root `_Pods.xcodeproj` is a symlink to `Example/Pods/Pods.xcodeproj` (created after `pod install`).
